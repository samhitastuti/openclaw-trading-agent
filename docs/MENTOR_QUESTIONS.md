# Mentor Questions

## 10 Critical Questions for Design Validation

Use these questions when meeting with mentors to get the most actionable feedback in the shortest time.

---

1. **What would you cut to keep this MVP strong and judge-ready?**
   _(Focus the mentor on scope triage — what can we drop without losing the core value?)_

2. **Which part of our architecture is most risky right now?**
   _(Layer 1 AI reliability? Layer 2 policy completeness? Layer 3 Alpaca integration stability?)_

3. **What metric would best prove impact for this problem?**
   _(We currently use "100 % block rate on policy violations" — is there a stronger or more compelling metric?)_

4. **If you were a judge, what would be your first criticism?**
   _(Identify the weakest point before the judges do.)_

5. **What one UX change would most improve demo clarity?**
   _(Even a backend-only API — what would make the demo easier to follow for non-technical judges?)_

6. **Are we over-engineering any module for hackathon scope?**
   _(Is the 5-check enforcement chain overkill? Should we simplify the YAML rule structure?)_

7. **What security or privacy gap should we close before demo?**
   _(Beyond the secrets-in-repo fix — what else might a security-focused judge flag?)_

8. **How can we present this better for non-technical judges?**
   _(The 3-layer model is technical — what's the simplest analogy or visual that makes it click?)_

9. **What should be our strongest differentiation statement?**
   _(Compared to "just calling the Alpaca API with GPT" — what's our key defensible advantage?)_

10. **What can we do in the next 2 hours for maximum score gain?**
    _(Time-box the mentor's answer — force them to prioritise the highest-leverage action.)_
